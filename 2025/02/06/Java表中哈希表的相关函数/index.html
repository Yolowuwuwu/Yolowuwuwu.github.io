小编近期在练习哈希表时，学习到很多相关的函数，特此总结一下。

参考来源：菜鸟教程

1. Java HashSet

HashSet 是 Java 集合框架中的一个实现了 Set 接口的类，它基于哈希表来存储元素。Set 是一种不允许重复元素的集合，而 HashSet 作为 Set 的一种实现，具有以下特点和行为：

1.1 特点

1.1.1 无序性：

HashSet 内部元素的顺序是不确定的，它不保证元素的存储顺序。你插入的元素可能并不按插入的顺序取出。

1.1.2 不重复：

HashSet 不允许重复的元素。如果你试图添加一个已经存在的元素，它将不会被添加。

1.1.3 性能：

HashSet 的元素查找、插入和删除操作的时间复杂度一般为 O(1)，但这依赖于哈希函数的性能以及哈希表的负载因子。

1.2 HashSet 的创建

HashSet 中的元素实际上是对象，一些常见的基本类型可以使用它的包装类。

基本类型对应的包装类表如下：

HashSet 类位于 java.util 包中，使用前需要引入它，语法格式如下：

import java.util.HashSet; // 引入 HashSet 类

以下实例我们创建一个 HashSet 对象 sites，用于保存字符串元素：

HashSet result= new HashSet<>();

1.3 HashSet 常见用法

1.3.1 添加元素 -> add()

result.add(1);
result.add(2);
result.add(1);

输出：[1,2]
虽然数字“1”被添加了两次，但是最后只会存一个1，因为HashSet是不可重复的。

1.3.2 判断元素是否存在 -> contains()

System.out.println(result.contains(1));
System.out.println(result.contains(3));

输出：
true
false

1.3.3 删除元素 -> remove()

result.remove(2);

1.3.4 清除所有元素 ->clear()

result.clear();

1.3.5 计算大小 -> size()

result.size();

1.3.6 迭代 -> for-each

for(Interget i:result)

2. Java HashMap

HashMap 是 Java 集合框架中一个非常常用的实现了 Map 接口的类，它基于哈希表（HashTable）实现，用于存储键值对（key-value pair）。HashMap 是一个无序的集合，不保证元素的顺序，键和值可以为 null，且键是唯一的。

2.1 特点

2.1.1 存储键值对：

HashMap 存储的是一对一的映射关系，每个键都对应一个值，且每个键是唯一的。

2.1.2 无序性：

HashMap 内部不保证存储顺序，它是通过哈希函数将元素存储在不同的桶中。

2.1.3 键唯一：

HashMap 中的键是唯一的，如果重复插入同一个键，会更新该键对应的值，而不是插入重复的键值对。

2.1.4 支持 null 值：

HashMap 允许键和值都为 null，但每个键只能出现一次，因此只能有一个 null 键。

2.1.5 性能：

HashMap 提供常数时间复杂度（O(1)）的查找、插入和删除操作，前提是哈希函数比较均匀，没有过多的哈希冲突。

1.2 HashMap 的创建

HashMap 中的元素实际上是对象，一些常见的基本类型可以使用它的包装类。

基本类型对应的包装类表如下：

HashMap 类位于 java.util 包中，使用前需要引入它，语法格式如下：

import java.util.HashMap; // 引入 HashMap 类

以下实例我们创建一个 HashMap 对象 Sites， 整型（Integer）的 key 和字符串（String）类型的 value：

HashMap<Integer, String> Sites = new HashMap<Integer, String>();

再创建一个HashMap对象map，整型（Integer）和整型（Integer）的value：

HashMap<Integer, Integer> Sites = new HashMap<>();
